Vigil

Your agent calls tools.
You should know what it proposed.

Vigil is a deterministic trace layer for probabilistic tool-use behaviour. Every tool proposal your agent makes gets normalized into a structured, auditable record — before anything executes.

The problem

When an agent calls a tool, you get a log entry. Useful for debugging one thing. But if you want to understand what your agent is doing across many calls — what it consistently proposes, what it gets wrong, what a regulator would see — a log entry is not enough. The raw output is messy, inconsistent, and hard to compare.

What Vigil does
01 —
normalize
Messy tool proposal goes in. A deterministic structured record comes out. Stable anchor fields. Comparable across calls.
02 —
review
Judge the record with no side effects. Get a governed verdict back. Nothing executes. Everything is auditable.
03 —
store
Every call is stored. Raw payload preserved. Normalized layer on top. Review results attached when present. The full chain is yours to keep.
Quickstart — normalize a tool proposal
curl -X POST https://your-endpoint/gateway/v0/normalize \
  -H "X-Vigil-Api-Key: <your-beta-key>" \
  -H "Content-Type: application/json" \
  -d '{'
    "actor_id":     "agent:my-agent",
    "principal_id": "usr:developer",
    "source_ref":   "session:abc123",
    "proposal": {
      "action_type":  "task.transition",
      "connector_id": "task-state-v1",
      "payload": {
        "task_id":    "T-42",
        "from_state": "pending",
        "to_state":   "in_review"
      }
    }
  }'

# returns →
{
  "request_id":   "req:abc...",
  "record_id":    "rec:abc...",
  "run_id":       "run:xyz...",
  "normalized_record": {...},
  "action_envelope": {...},
  "mode":         "record_only"
}
What comes back
normalized record
Stable anchor fields extracted from the messy proposal. The same action class always looks the same. Comparable across thousands of calls.
governed verdict
Returned by review, not normalize. A governed outcome and verdict with reasons attached, with no side effects.
raw payload preserved
The original proposal is stored alongside the normalized form. Nothing is thrown away. The audit trail is complete.
record_id chain
Request, record, run, outcome, and verdict stay linked in one audit chain. The normalized record becomes the stable reference point.
